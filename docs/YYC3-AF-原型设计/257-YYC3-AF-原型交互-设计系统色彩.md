### 4. 主题策略

#### 4.1 主题切换机制

系统使用CSS变量（自定义属性）支持即时、无闪烁的主题切换。

#### 4.2 主题变量映射

| Token | 浅色模式值 | 深色模式值 | 说明 |
|---|---|---|---|
| `--background` | `#f7fafc`（冰白） | `#050505`（赛博黑） | 背景色 |
| `--foreground` | `#1a365d`（深蓝） | `#cbd5e0`（柔灰） | 前景色 |
| `--primary` | `#1a365d`（深蓝） | `#3182ce`（科技蓝） | 主色 |
| `--border` | `#cbd5e0`（灰300） | `#2d3748`（灰800） | 边框色 |

#### 4.3 主题切换实现

```css
:root {
  --background: #f7fafc;
  --foreground: #1a365d;
  --primary: #1a365d;
  --border: #cbd5e0;
}

[data-theme="dark"] {
  --background: #050505;
  --foreground: #cbd5e0;
  --primary: #3182ce;
  --border: #2d3748;
}
```

### 5. 无障碍标准

#### 5.1 WCAG 2.1 AA合规性

- **对比度要求**：所有文本配对（背景 vs 前景）的对比度至少达到4.5:1
- **大文本对比度**：大文本（18pt及以上或14pt粗体）的对比度至少达到3:1
- **非文本对比度**：用户界面组件和图形对象的对比度至少达到3:1

#### 5.2 色彩独立性

- 色彩绝不作为传达信息的唯一视觉提示
- 状态颜色始终伴随图标或文本标签
- 支持色盲用户的信息获取

#### 5.3 焦点指示器

- 所有交互元素都有清晰的焦点指示器
- 焦点指示器使用高对比度颜色
- 焦点指示器尺寸至少为2px
