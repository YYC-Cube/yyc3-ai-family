### 3. 导航深入：从宏观到微观

虽然 Level 1-3 处理平台的结构导航，但 **Level 4（资源单元）** 和 **Level 5（详情属性）** 是实际操作工作发生的地方。这些层级是动态的、上下文相关的、数据驱动的。

### 4. Level 4: 资源单元导航（"纳米"视图）

#### 4.1 设计目标

提供在由 Level 3 上下文标识的特定资源集合内的导航。

#### 4.2 上下文行为

- **在 L04 AI Intelligence 中**：代表特定 Agent（Navigator、Thinker、Oracle）的标签页或卡片
- **在 L01 Infrastructure 中**：特定节点、Pod 或集群的列表或网格
- **在 L05 Business 中**：活动项目或工作流的列表

#### 4.3 UI 模式

- **分屏视图**：出现在 L3 边栏右侧的辅助列表面板
- **卡片网格**：主内容区域显示资源卡片网格（如当前 ConsoleView 所见）
- **搜索/过滤栏**：针对此特定资源集的本地化搜索

#### 4.4 示例：AI Intelligence Layer（L4）

- **导航项目**：
    1. **Agent: Navigator**（状态：空闲）
    2. **Agent: Thinker**（状态：思考中）
    3. **Agent: Oracle**（状态：离线）
    4. **Vector DB: Main**（状态：就绪）

### 5. Level 5: 详情属性导航（"皮科"视图）

#### 5.1 设计目标

对 Level 4 中选定的单个资源单元进行深度内省。

#### 5.2 上下文行为

- 当点击特定资源（例如 "Agent: Navigator"）时激活
- 替换主内容区域或打开滑过面板

#### 5.3 UI 模式

- **内部标签页**：位于资源视图内
- **标准标签页**：
  - `Overview`：健康状态、关键指标、摘要
  - `Config`：YAML/JSON 配置编辑器
  - `Logs`：实时流式日志
  - `Metrics`：历史性能图表
  - `Terminal`：SSH/Exec 控制台（用于 L1/L4 资源）
  - `Trace`：分布式追踪（用于 L3/L5 资源）

#### 5.4 示例：Agent: Navigator（L5 视图）

- **标签页 1: Status**："当前任务：分析代码库..."
- **标签页 2: Conversation**：与该 Agent 的聊天历史
- **标签页 3: Memory**：查看 RAG 上下文和短期记忆
- **标签页 4: Settings**：调整温度、模型（GPT-4/Claude-3）和提示词

### 6. 交互流程示例

1. **用户点击 L1 "System"**：边栏显示 L1-L9 层级
2. **用户点击 L3 "L01 Infrastructure"**：主区域显示 K8s 集群
3. **用户点击 L4 "Cluster-Alpha"**：主区域聚焦于 Cluster-Alpha
4. **用户点击 L5 "Nodes" 标签页**：查看该集群中的节点列表
5. **用户点击特定节点**：打开详细的节点指标
