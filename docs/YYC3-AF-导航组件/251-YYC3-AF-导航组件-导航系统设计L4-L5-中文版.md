## 🧭 导航深度解析：从宏观到微观

虽然第 1-3 层处理平台的结构导航，但 **第 4 层（资源单元）** 和 **第 5 层（详细属性）** 是实际操作工作发生的地方。这些层级是动态的、上下文敏感的、数据驱动的。

---

### 🟡 第 4 层：资源单元导航（"纳米"视图）

**设计目标**：在由第 3 层上下文标识的特定资源集合内提供导航。

**上下文行为**：

* **在 L04 AI 智能层时**：代表特定智能体（Navigator、Thinker、Oracle）的标签页或卡片。
* **在 L01 基础设施层时**：特定节点、Pod 或集群的列表或网格。
* **在 L05 业务逻辑层时**：活动项目或工作流的列表。

**UI 模式**：

* **分屏视图**：出现在 L3 侧边栏右侧的辅助列表面板。
* **卡片网格**：主内容区域显示资源卡片网格（如当前 ConsoleView 所示）。
* **搜索/过滤栏**：针对此特定资源集的本地化搜索。

**示例：AI 智能层（L4）**

* *导航项*：
    1. **智能体：Navigator**（状态：空闲）
    2. **智能体：Thinker**（状态：思考中）
    3. **智能体：Oracle**（状态：离线）
    4. **向量数据库：Main**（状态：就绪）

---

### 🔴 第 5 层：详细属性导航（"皮米"视图）

**设计目标**：对第 4 层中选定的单个资源单元进行深度内省。

**上下文行为**：

* 当点击特定资源（例如"智能体：Navigator"）时激活。
* 替换主内容区域或打开滑出面板。

**UI 模式**：

* **内部标签页**：位于资源视图内。
* **标准标签页**：
  * `概览`：健康状态、关键指标、摘要。
  * `配置`：YAML/JSON 配置编辑器。
  * `日志`：实时流式日志。
  * `指标`：历史性能图表。
  * `终端`：SSH/Exec 控制台（用于 L1/L4 资源）。
  * `追踪`：分布式追踪（用于 L3/L5 资源）。

**示例：智能体：Navigator（L5 视图）**

* **标签页 1：状态**："当前任务：分析代码库..."
* **标签页 2：对话**：与该智能体的聊天历史。
* **标签页 3：记忆**：查看 RAG 上下文和短期记忆。
* **标签页 4：设置**：调整温度、模型（GPT-4/Claude-3）和提示词。

## 🔄 交互流程示例

1. **用户点击 L1 "系统"**：侧边栏显示 L1-L9 层级。
2. **用户点击 L3 "L01 基础设施"**：主区域显示 K8s 集群。
3. **用户点击 L4 "Cluster-Alpha"**：主区域聚焦于 Cluster-Alpha。
4. **用户点击 L5 "节点"标签页**：查看该集群中的节点列表。
5. **用户点击特定节点**：打开详细的节点指标。

# YYC3 分层自治单元导航栏设计 (Layered Autonomous Unit Navigation)

## 🧭 导航架构设计原则

基于 **九层功能架构体系** 与 **五高五标五化** 核心机制，本设计旨在构建一个清晰、高效、可扩展的导航系统，实现对庞大 DevOps 平台的精准掌控。

### 1. 核心原则

| 原则 | 描述 |
| :--- | :--- |
| **层级映射 (Mapping)** | 导航结构严格镜像架构层级 |
| **全息可视 (Holographic)** | 导航不仅是入口，更是状态监控器（如红点、进度条） |
| **情境感知 (Contextual)** | 根据当前选中的层级，动态展示相关工具与视图 |

---

### 2. 五级导航体系 (Five-Level Navigation System)

#### 🟢 第 1 层：全局功能锚点 (Global Anchors)

**定位**：系统左侧一级侧边栏，提供顶级业务域的快速切换。

| 图标 | 名称 | 对应架构域 | 功能描述 |
| :--- | :--- | :--- | :--- |
| 🏠 | **主页** | Portal | 综合概览与快捷入口 |
| 📊 | **仪表板** | Visual | 全局关键指标监控 |
| 🤖 | **AI 智能中心** | Layer-04 | 智能体管理与模型训练 |
| 💼 | **业务管理** | Layer-05 | 项目管理与业务流程 |
| 🗄️ | **数据中心** | Layer-02 | 数据库管理与存储概览 |
| ⚙️ | **系统设置** | Layer-01/09 | 基础设施与系统设置 |

---

#### 🔵 第 2 层：功能视图切片 (Functional Perspectives)

**定位**：顶部二级标签栏，提供当前上下文的不同维度视图。

| 视图 | 描述 |
| :--- | :--- |
| **📐 架构视图** | 拓扑结构与层级关系 |
| **🔍 搜索** | 全局检索与代码搜索 |
| **📚 文档** | 架构文档与 API 说明 |
| **🧪 测试** | 单元测试与集成测试状态 |
| **📈 监控** | 实时性能监控与告警 |

---

#### 🟣 第 3 层：架构层级树 (Architecture Layer Tree)

**定位**：左侧二级侧边栏（可折叠），提供九层架构的垂直钻取能力。

| 层级 | 名称 | 状态 |
| :--- | :--- | :--- |
| **Layer-09** | 系统设置 (System Settings) | ✅ 激活 |
| **Layer-08** | 扩展演进 (Extension & Evolution) | ✅ 激活 |
| **Layer-07** | 用户交互 (User Interaction) | ✅ 激活 |
| **Layer-06** | 应用表现 (App Presentation) | ✅ 激活 |
| **Layer-05** | 业务逻辑 (Business Logic) | ✅ 激活 |
| **Layer-04** | AI 智能 (AI Intelligence) | ✅ 激活 |
| **Layer-03** | 核心服务 (Core Services) | ✅ 激活 |
| **Layer-02** | 数据存储 (Data Storage) | ✅ 激活 |
