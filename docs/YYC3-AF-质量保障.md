# YYC³ AI Family - Figma AI 项目质量保障提示词要求

## 🎯 总体设计目标

基于 YYC³ 九层架构体系和五维实施规划，设计一个**智能、高效、一致、可扩展**的设计系统，确保 UI/UX 设计符合"五高五标五化"原则。

## 一、核心设计原则要求

### 1.1 五高设计原则

```
🤖 高智能设计 (High Intelligence Design):
- AI 驱动的交互：界面能根据用户行为智能调整布局和功能推荐
- 自适应内容：组件和布局根据数据内容和上下文自动优化
- 预测性交互：提前预测用户需求并提供快捷操作
- 智能可视化：数据图表根据数据类型自动选择最佳可视化方式
- 自然语言界面：支持自然语言搜索和命令执行

📱 高体验设计 (High Experience Design):
- 响应式断点：移动端(375px)、平板(768px)、桌面(1280px)、大屏(1920px+)
- 触摸友好：移动端触摸目标最小 44x44px，间距合理
- 手势操作：支持滑动、长按、双指缩放等自然手势
- 加载状态：骨架屏、进度指示器、渐进式加载
- 错误处理：优雅的错误提示和恢复机制

🚀 高性能设计 (High Performance Design):
- 轻量级组件：组件代码和资源文件最小化
- 懒加载策略：图片、组件、模块按需加载
- 动画优化：使用 transform 和 opacity 实现 60fps 流畅动画
- 字体优化：系统字体栈，字体文件压缩和子集化
- 图片优化：WebP 格式，响应式图片，懒加载

🔒 高安全设计 (High Security Design):
- 敏感操作确认：删除、修改、导出等操作需要二次确认
- 权限可视化：当前用户权限状态清晰可见
- 安全状态指示：加密状态、连接安全状态视觉提示
- 访问控制：不同权限用户看到不同的界面元素
- 审计追踪：操作记录可视化展示

🌱 高扩展设计 (High Scalability Design):
- 模块化组件：组件可独立使用和组合
- 主题系统：支持多主题和自定义主题
- 国际化：支持多语言和地区格式
- 插件架构：界面支持插件扩展
- 配置化：界面元素和布局可配置
```

### 1.2 五标设计标准

```
🎨 视觉标准 (Visual Standards):
- 设计令牌系统：颜色、间距、字体、圆角等统一使用设计令牌
- 组件变体：每个组件有明确的状态变体（默认、悬停、激活、禁用、加载等）
- 图标系统：统一图标风格和大小，Figma 组件库管理
- 动效标准：统一的动画时长、缓动函数、过渡效果

📏 布局标准 (Layout Standards):
- 8px 网格系统：所有间距和尺寸使用 8 的倍数
- 响应式网格：12列网格系统，适配不同屏幕
- 间距比例：使用 1:1.618 黄金比例确定间距层级
- 对齐规则：左对齐文本，数字右对齐，图标与文字中心对齐

📝 内容标准 (Content Standards):
- 文案规范：统一语气（专业、友好）、术语、缩写规则
- 错误信息：有指导性，包含解决方案或联系支持的方式
- 帮助文本：表单字段有清晰的说明和示例
- 空状态：有意义的空状态设计和引导操作

🔌 交互标准 (Interaction Standards):
- 焦点管理：键盘导航顺序符合逻辑，焦点状态清晰可见
- 反馈机制：操作后有明确的视觉或触觉反馈
- 手势映射：统一的手势与操作映射关系
- 快捷键：统一快捷键设置，支持自定义

📊 数据标准 (Data Standards):
- 数据格式化：数字、日期、货币的统一格式化
- 图表规范：统一的图表颜色、标签、图例样式
- 加载状态：数据加载中的统一占位符设计
- 错误状态：数据错误或空状态的统一处理方式
```

### 1.3 五化设计策略

```
🤖 智能化设计 (Intelligent Design):
- 智能推荐：根据用户历史和行为推荐相关功能和内容
- 自适应布局：根据内容量和屏幕尺寸自动调整布局
- 预测性输入：表单输入智能补全和验证
- 上下文感知：界面根据用户当前任务和环境调整

⚡ 自动化设计 (Automated Design):
- 设计令牌：颜色、字体、间距等使用设计令牌自动同步
- 组件库：统一的组件库，自动更新所有实例
- 设计系统文档：自动生成设计规范文档
- 响应式适配：设计稿自动适配不同屏幕尺寸

👁️ 可视化设计 (Visualized Design):
- 数据可视化：复杂的系统状态和数据用可视化方式呈现
- 流程可视化：工作流程和状态流转可视化展示
- 关系可视化：实体间的关系网络图
- 时间线可视化：时间序列数据的可视化展示

🧩 模块化设计 (Modular Design):
- 原子设计：原子 → 分子 → 有机体 → 模板 → 页面
- 组件变体：组件有明确的变体和使用场景说明
- 模式库：常用交互模式和布局模式的库
- 模板系统：常用页面模板，快速构建新页面

🌐 生态化设计 (Ecological Design):
- 插件系统：界面支持第三方插件集成
- API 可视化：API 文档和测试的可视化界面
- 开发者工具：开发者专用的调试和配置界面
- 社区功能：用户反馈、建议、分享功能
```

### 4.6 动效系统 (Animation System)

```

- 中速: 300ms (页面过渡)
- 慢速: 500ms (复杂动画)
- 极慢: 1000ms (引导动画)

🔄 缓动函数 (Easing Functions):
- 线性: linear (无缓动)
- 标准: cubic-bezier(0.4, 0, 0.2, 1) (默认)
- 进入: cubic-bezier(0, 0, 0.2, 1) (元素进入)
- 退出: cubic-bezier(0.4, 0, 1, 1) (元素退出)
- 弹性: cubic-bezier(0.68, -0.55, 0.265, 1.55) (有趣的效果)

🎬 动画模式 (Animation Patterns):
- 淡入淡出: opacity + translateY
- 滑动: translateX 或 translateY
- 缩放: scale
- 旋转: rotate
- 组合动画: 多个属性同时变化

📱 性能优化 (Performance Optimization):
- 优先使用 transform 和 opacity
- 避免动画期间重排
- 使用 will-change 提示浏览器
- 减少同时动画的元素数量
```

```
   "生成用户从登录到完成一个 AI 任务的完整流程。包含以下步骤：登录页面 → 主仪表盘 → AI 聊天界面 → 任务配置 → 执行结果 → 导出结果。展示关键交互状态和错误处理场景。"

5. 生成响应式设计:
   "为 AI 模型管理界面生成响应式设计。桌面版显示完整的表格和侧边栏，平板版显示简化表格和底部导航，移动版显示卡片列表和底部导航。展示三个断点：375px（移动端）、768px（平板端）、1280px（桌面端）。"
```

### 7.2 设计检查提示词

```
当使用 Figma AI 检查设计时，请检查以下方面:

1. 设计系统一致性检查:
   "检查这个设计文件是否完全遵循 YYC³ 设计系统。特别检查：颜色是否使用设计令牌，字体是否使用字体规模，间距是否使用间距系统，组件是否使用标准组件。"

2. 无障碍检查:
   "检查这个设计是否满足无障碍要求。特别检查：颜色对比度是否达到 WCAG 2.1 AA 标准，焦点状态是否清晰可见，交互元素是否有足够的触摸目标，文本是否可调整大小。"

3. 响应式检查:
   "检查这个设计是否满足响应式要求。特别检查：不同断点是否有相应的布局调整，触摸目标和字体大小是否适合移动设备，复杂组件在移动端是否有简化版本。"

4. 用户体验检查:
   "检查这个设计的用户体验问题。特别检查：用户流程是否顺畅，错误处理是否合理，加载状态是否有适当反馈，复杂操作是否有引导或帮助。"

5. 设计细节检查:
   "检查这个设计的细节问题。特别检查：对齐是否准确，间距是否一致，图标风格是否统一，阴影和圆角是否遵循规范。"
```

### 7.3 设计优化提示词

```
当使用 Figma AI 优化设计时，请考虑以下优化方向:

1. 性能优化:
   "优化这个设计以减少前端实现时的性能问题。建议：减少不必要的阴影和渐变，简化复杂动画，优化图片使用，考虑懒加载策略。"

2. 开发友好优化:
   "优化这个设计以方便前端开发。建议：使用标准的组件变体，提供清晰的设计令牌，避免复杂的绝对定位，提供响应式规则。"

3. 用户体验优化:
   "优化这个设计以提升用户体验。建议：添加空状态设计，优化错误提示，添加加载状态，提供快捷操作，简化复杂流程。"

4. 无障碍优化:
   "优化这个设计以提升无障碍体验。建议：提高颜色对比度，添加焦点状态，提供替代文本，优化键盘导航。"

5. 视觉层次优化:
   "优化这个设计的视觉层次。建议：通过颜色、大小、间距建立清晰的视觉层次，突出重要内容，弱化次要内容，确保用户视线流合理。"
```
