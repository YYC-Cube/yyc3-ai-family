# 开发工具配置参考

> GLM Coding Plan 支持 15+ 主流编程工具，本文件提供详细的配置指南。

## 工具分类

### 终端工具

| 工具 | 特点 | 适用场景 |
|-----|-------|---------|
| Claude Code | 终端智能编码，自然语言交互 | 日常编码、调试、重构 |
| OpenCode | 终端 + IDE 编程代理 | 多语言支持、项目管理 |
| Crush | CLI + TUI 界面 | 命令行编程、多任务处理 |
| OpenClaw | 本地 AI 助手 | 多平台消息、自动化任务 |

### VS Code 插件

| 工具 | 特点 | 适用场景 |
|-----|-------|---------|
| Cline | VS Code AI 插件，支持 MCP | 代码生成、文件操作 |
| Roo Code | VS Code 项目分析助手 | 项目重构、代码审查 |
| Kilo Code | VS Code 代码生成插件 | 快速编码、代码补全 |
| Cursor | AI 驱动的代码编辑器 | 实时协作、智能补全 |

### IDE 集成

| 工具 | 特点 | 适用场景 |
|-----|-------|---------|
| TRAE | Trae IDE 原生集成 | 全栈开发、团队协作 |
| Cherry Studio | 代码生成 IDE | 快速原型、多语言 |
| Factory-Droid | Android 开发工具 | 移动应用开发 |

### AI Agent 工具

| 工具 | 特点 | 适用场景 |
|-----|-------|---------|
| Goose | AI Agent 工具，支持 MCP | 自动化任务、智能决策 |

## 通用配置信息

### API 端点

所有 Coding Plan 工具使用统一端点：

```
https://open.bigmodel.cn/api/coding/paas/v4
```

### 模型选择

| 模型 | 用途 | 特点 |
|-----|-------|------|
| GLM-5 | 复杂任务 | 参数量最大，对标 Opus |
| GLM-4.7 | 通用编码 | 平衡性能与成本 |
| GLM-4.5-Air | 轻量任务 | 响应快速，成本最低 |

<Tip>
  GLM-5 在高峰期（14:00-18:00 UTC+8）按 3 倍扣费，非高峰期按 2 倍扣费。
  建议复杂任务使用 GLM-5，普通任务使用 GLM-4.7。
</Tip>

## 详细配置

### Claude Code

#### 环境变量配置

**macOS/Linux**
```bash
export ANTHROPIC_AUTH_TOKEN="your_api_key"
export ANTHROPIC_BASE_URL="https://open.bigmodel.cn/api/anthropic"
```

**Windows (PowerShell)**
```powershell
[System.Environment]::SetEnvironmentVariable('ANTHROPIC_AUTH_TOKEN', 'your_api_key', 'User')
[System.Environment]::SetEnvironmentVariable('ANTHROPIC_BASE_URL', 'https://open.bigmodel.cn/api/anthropic', 'User')
```

**配置文件 ~/.claude/settings.json**
```json
{
  "env": {
    "ANTHROPIC_AUTH_TOKEN": "your_api_key",
    "ANTHROPIC_BASE_URL": "https://open.bigmodel.cn/api/anthropic",
    "API_TIMEOUT_MS": "3000000"
  }
}
```

#### 模型映射配置（使用 GLM-5）

```json
{
  "env": {
    "ANTHROPIC_DEFAULT_HAIKU_MODEL": "glm-4.5-air",
    "ANTHROPIC_DEFAULT_SONNET_MODEL": "glm-4.7",
    "ANTHROPIC_DEFAULT_OPUS_MODEL": "glm-5"
  }
}
```

### Cline

#### VS Code 配置

```json
{
  "mcpServers": {
    "zai-mcp-server": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@z_ai/mcp-server"],
      "env": {
        "Z_AI_API_KEY": "your_api_key",
        "Z_AI_MODE": "ZHIPU"
      }
    }
  },
  "apiProvider": "Z AI",
  "zaiEntrypoint": "China Coding Plan (https://open.bigmodel.cn/api/coding/paas/v4)",
  "zaiApiKey": "your_api_key",
  "model": "glm-5"
}
```

### Cursor

#### Custom Model 配置

1. 打开 Cursor
2. 进入 Models -> Add Custom Model
3. 配置：
   - **Protocol**: OpenAI
   - **API Key**: your_api_key
   - **Override OpenAI Base URL**: https://open.bigmodel.cn/api/coding/paas/v4
   - **Model Name**: GLM-5 / GLM-4.7 / GLM-4.5-Air

## 常见配置问题

### API Key 鉴权失败

**问题**: 401 Unauthorized 错误

**解决方案**:
1. 检查 API Key 是否正确复制
2. 确认 API Key 来自 https://bigmodel.cn/usercenter/proj-mgmt/apikeys
3. 检查是否使用了正确的 API 端点

### 工具无法连接

**问题**: 无法连接到 API 端点

**解决方案**:
1. 检查网络连接
2. 确认使用的端点为 https://open.bigmodel.cn/api/coding/paas/v4
3. 尝试重启工具

### 模型不显示

**问题**: 配置后看不到 GLM 模型

**解决方案**:
1. 检查配置文件格式是否正确
2. 确认已订阅 Coding Plan
3. 查看工具是否支持自定义模型
