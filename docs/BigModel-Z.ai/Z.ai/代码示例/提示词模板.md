## 最佳实践

### 提示词编写原则

1. **明确具体**
   - 清晰表达需求
   - 避免模糊表述
   - 提供具体示例

2. **上下文丰富**
   - 提供足够的背景信息
   - 说明使用场景
   - 列出技术约束

3. **结构化输出**
   - 指定期望的输出格式
   - 分点列出要求
   - 提供模板示例

4. **分步引导**
   - 复杂任务分解步骤
   - 逐步验证结果
   - 及时调整方向

### 使用技巧

1. **角色扮演**
   ```
   你现在是一位资深的 [角色]，请基于你的专业视角帮我...
   ```

2. **思维链**
   ```
   请一步步思考这个问题：
   1. 分析问题
   2. 拆解任务
   3. 制定方案
   4. 实施验证
   ```

3. **少样本学习**
   ```
   请按照以下格式回答：

   示例1：
   输入：[示例输入1]
   输出：[示例输出1]

   示例2：
   输入：[示例输入2]
   输出：[示例输出2]

   现在，请回答：
   输入：[实际输入]
   输出：
   ```

4. **约束条件**
   ```
   请编写代码，但请注意以下约束：
   - 约束1
   - 约束2
   - 约束3
   ```
